💡 Best Practices

✅ Foreign Key ব্যবহার করো।

✅ Relationship অনুযায়ী Table তৈরি করো।

✅ সব Table-এ created_at এবং updated_at রাখো।

✅ slug ব্যবহার করো SEO-friendly URL-এর জন্য।

✅ status ফিল্ড রাখো (Publish/Draft, Active/Inactive)।

⚠️ Common Mistakes

❌ Foreign Key না ব্যবহার করা।

❌ instructor_name বা category_name সরাসরি Course Table-এ রাখা (এর বদলে instructor_id, category_id ব্যবহার করবে)।

❌ Relationship না ভেবে Table তৈরি করা।

❌ একই তথ্য একাধিক Table-এ রাখা (Data Duplication)।

🎤 Interview Questions
Q1: Foreign Key কী?

Answer:
Foreign Key একটি Table-কে অন্য Table-এর সাথে সম্পর্ক (Relationship) তৈরি করতে সাহায্য করে এবং Data Integrity বজায় রাখে।

Q2: Course Table-এ category_name না রেখে category_id কেন রাখি?

Answer:
কারণ এতে Database Normalize থাকে, Data Duplicate হয় না, এবং Category-এর নাম পরিবর্তন করলে এক জায়গায় পরিবর্তন করলেই সব জায়গায় Update হয়ে যায়।

Q3: Course এবং Lesson-এর মধ্যে সরাসরি Relationship না করে Section কেন ব্যবহার করেছি?

Answer:
কারণ একটি বড় Course-কে বিভিন্ন অধ্যায় (Section)-এ ভাগ করলে Content Management সহজ হয় এবং User Experience ভালো হয়।

Q1

Draft এবং Published Status কেন ব্যবহার করা হয়?

Answer

কারণ Course ছাড়া Section, Lesson, Enrollment, Quiz, Certificate—কোনো Module কাজ করবে না।

Q1

Notification এবং Announcement-এর মধ্যে পার্থক্য কী?

Answer

Notification সাধারণত নির্দিষ্ট User-এর জন্য এবং Event-based। Announcement অনেক User-এর জন্য একটি Public Information।

Read / Unread Status কেন দরকার?

Answer

User কোন Notification দেখেছে আর কোনটি দেখেনি তা Track করার জন্য।

Notification Module-এর সবচেয়ে গুরুত্বপূর্ণ Security কী?

Answer

User Authorization, Target User Validation এবং Secure Redirect URL।
